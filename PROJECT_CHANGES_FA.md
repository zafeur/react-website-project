# گزارش تغییرات انجام‌شده در پروژه

## ۱. تبدیل پروژه به Next.js

پروژه از حالت React معمولی/Vite به ساختار Next.js منتقل شد. برای این کار ساختار routing پروژه تغییر کرد و صفحات اصلی داخل پوشه `pages` قرار گرفتند.

فایل‌های اصلی Next.js:

- `pages/_app.jsx`: نقطه اصلی ورود استایل‌ها در Next.js. تمام فایل‌های CSS عمومی و مربوط به کامپوننت‌ها از اینجا import شدند.
- `pages/index.jsx`: صفحه اصلی سایت.
- `pages/restaurant.jsx`: صفحه رستوران.
- `pages/dashboard.jsx`: صفحه داشبورد کاربر.
- `pages/faq.jsx`: صفحه سوالات متداول و فرم عضویت.
- `next.config.js`: فایل تنظیمات Next.js.
- `package.json`: اسکریپت‌ها به دستورات Next تغییر کردند.

اسکریپت‌های اصلی پروژه:

```json
{
  "dev": "next dev",
  "build": "next build",
  "start": "next start"
}
```

## ۲. ساخت و اصلاح صفحه اصلی

صفحه اصلی طبق فایل‌های HTML و CSS قبلی ساخته شد، اما با ظاهر و تم کلی صفحات داخلی هماهنگ شد.

کارهای انجام‌شده:

- ساخت هدر هماهنگ با باقی صفحات
- هماهنگ کردن لوگو و رنگ‌بندی
- اضافه کردن لینک‌ها به صفحات دیگر مثل `/restaurant` و `/faq`
- تبدیل بنرهای صفحه اصلی به اسلایدر
- اضافه کردن دکمه‌های next و previous
- اضافه کردن drag/swipe برای موبایل و دسکتاپ
- توقف تایمر اسلایدر هنگام نگه داشتن موس یا لمس
- برگشت از آخرین اسلاید به اولین اسلاید و برعکس
- اصلاح اندازه و جایگیری تصاویر
- داینامیک کردن دیتای صفحه اصلی با fallback data

فایل‌های مربوط:

- `src/components/HomePage.jsx`
- `src/components/HomePage.css`
- `src/api/home.js`

## ۳. صفحه سوالات متداول و فرم عضویت

یک صفحه جدید برای سوالات متداول ساخته شد. در این صفحه ابتدا توضیحات و سوالات متداول نمایش داده می‌شود، سپس فرم عضویت قرار دارد.

فرم عضویت دو مرحله‌ای است:

- مرحله اول: اطلاعات مجموعه
- مرحله دوم: اطلاعات کسب‌وکار و اطلاعات مارکتینگ

فرم به صورت اسلایدی بین دو مرحله حرکت می‌کند و اطلاعات به سمت بک‌اند ارسال می‌شود.

فایل‌های مربوط:

- `pages/faq.jsx`
- `src/components/FaqMembershipPage.jsx`
- `src/components/FaqMembershipPage.css`
- `src/api/membership.js`

## ۴. اصلاح ساختار API و Helper

ساختار قبلی این مشکل را داشت که فایل‌های داخل `api` برای ابزارهای عمومی مثل Axios و Cookie هم استفاده می‌شدند. این ساختار اصلاح شد.

الان پوشه `api` فقط برای درخواست‌های مربوط به سرور استفاده می‌شود.

فایل‌های `api`:

- `src/api/auth.js`: برای درخواست‌های مربوط به ورود، ارسال OTP و تایید OTP.
- `src/api/membership.js`: برای ارسال فرم عضویت.
- `src/api/home.js`: برای گرفتن دیتای داینامیک صفحه اصلی.

پوشه `helper` برای ابزارهای عمومی ساخته شد:

- `src/helper/httpClient.js`: تنظیمات عمومی Axios مثل `baseURL`، `headers`، `timeout` و interceptor.
- `src/helper/authMiddleware.js`: قبل از ارسال درخواست‌ها اجرا می‌شود و بررسی می‌کند که آیا درخواست نیاز به احراز هویت دارد یا نه.
- `src/helper/authCookie.js`: مدیریت توکن داخل cookie، شامل ذخیره، گرفتن و پاک کردن token.
- `src/helper/normalizeMobileNumber.js`: استانداردسازی شماره موبایل قبل از ارسال.
- `src/helper/apiBaseUrl.js`: مدیریت آدرس API سمت کلاینت و سرور.

## ۵. اصلاح احراز هویت و Cookie

ساختار ذخیره توکن تغییر کرد تا توکن داخل cookie ذخیره شود.

علت این کار:

- Cookie برای احراز هویت مناسب‌تر است.
- Middleware می‌تواند cookie را قبل از باز شدن صفحه بررسی کند.
- امکان مدیریت session بهتر می‌شود.

فایل مربوط:

- `src/helper/authCookie.js`

## ۶. اضافه شدن Middleware

برای محافظت از مسیرهایی مثل `/dashboard` فایل middleware اضافه شد.

فایل مربوط:

- `middleware.js`

کار middleware:

- قبل از باز شدن مسیرهای محافظت‌شده اجرا می‌شود.
- بررسی می‌کند آیا token داخل cookie وجود دارد یا نه.
- اگر token وجود نداشته باشد، کاربر به صفحه اصلی برمی‌گردد.
- اگر نوع کاربر مشخص باشد، بررسی می‌کند که user معتبر باشد.

نکته: Next.js نسخه جدید هشدار می‌دهد که در آینده بهتر است `middleware.js` با `proxy` جایگزین شود، اما در حال حاضر middleware کار می‌کند و build بدون خطا انجام شده است.

## ۷. ارسال اطلاعات با JSON

درخواست شده بود که دیگر از `FormData` و `append` استفاده نشود. بنابراین ارسال اطلاعات فرم‌ها به صورت JSON انجام شد.

مثال برای OTP:

```json
{
  "mobile": "09123456789"
}
```

برای فرم عضویت، اطلاعات در سه بخش ارسال می‌شود:

- `collection`
- `business`
- `marketing`

## ۸. اصلاح OTP

در بخش OTP چند اصلاح انجام شد:

- فایل `auth.js` مستقیم `axios` را import می‌کند.
- برای OTP از `request.js` استفاده نمی‌شود.
- شماره موبایل قبل از ارسال normalize می‌شود.
- ارسال OTP از مسیر Next API proxy انجام می‌شود.

مسیرهای proxy ساخته‌شده:

- `pages/api/auth/send-otp.js`
- `pages/api/auth/verify-otp.js`

علت اضافه شدن proxy:

درخواست مستقیم از مرورگر به `api.didarads.com` ممکن بود به خاطر CORS یا مشکلات شبکه fail شود. با proxy، مرورگر به خود پروژه Next درخواست می‌زند و Next از سمت سرور به بک‌اند وصل می‌شود.

با این حال مشخص شد که دامنه بک‌اند از این شبکه پاسخ نمی‌دهد:

```txt
api.didarads.com
```

DNS دامنه درست resolve می‌شود، اما ping و اتصال HTTPS timeout می‌شوند. بنابراین اگر OTP هنوز ارسال نشود، مشکل از سمت اتصال به بک‌اند یا محدودیت شبکه/سرور است، نه از UI فرانت.

## ۹. مدیریت خطاها

پیام خطای خام مثل زیر دیگر مستقیم به کاربر نمایش داده نمی‌شود:

```txt
timeout of 12000ms exceeded
```

در صورت timeout یا مشکل ارتباط با بک‌اند، پیام تمیزتر نمایش داده می‌شود:

```txt
خطا در ارتباط با سرور
```
